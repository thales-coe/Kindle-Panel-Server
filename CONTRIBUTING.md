# Contributing

## Padrão geral
- commits pequenos e objetivos;
- nomes claros para branches;
- sem versionar credenciais ou dados pessoais;
- toda mudança visual deve preservar legibilidade em tela e-ink.

## Convenção sugerida de branch
- `feat/<descricao>`
- `fix/<descricao>`
- `docs/<descricao>`
- `refactor/<descricao>`

## Convenção sugerida de commit
- `feat: adiciona bloco de clima`
- `fix: corrige parser de RSS`
- `docs: atualiza instrucoes de deploy`
- `refactor: separa funcoes de calendario`

## Pull requests
Todo PR deve responder:
1. o que mudou;
2. por que mudou;
3. se impacta performance no Kindle;
4. se introduz nova configuração local;
5. se exige atualização da documentação.
